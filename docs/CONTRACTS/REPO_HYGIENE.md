# REPO_HYGIENE Contract

## Purpose
Prevent repo drift, root clutter, and stale lock stalls before expensive autopilot work.

## Gate Script
- Script: `scripts/automation/repo_hygiene_gate.ps1`
- Modes:
  - `WARN` = report only (non-blocking)
  - `FAIL` = block on violations

## Untracked Allowlist Rules
Untracked files are only allowed under these prefixes:
- `artifacts/`
- `data/`
- `docs/`
- `memory/`

Anything untracked outside those prefixes is reported in `other`.

### Root cleanliness rule
Any untracked file at repo root is a `root-leaks` violation.

## Oragorn Session Dump Policy
Pattern generated by OpenClaw internal hook: `memory/YYYY-MM-DD-*.md`

Repo-local preflight adapter (authoritative):
- Script: `scripts/automation/route_oragorn_session_dumps.ps1`
- Called first by: `scripts/automation/repo_hygiene_gate.ps1`
- Behavior: routes Oragorn raw session dumps from
  - `C:\Users\Clamps\.openclaw\workspace-oragorn\memory\YYYY-MM-DD-*.md`
  to
  - `C:\Users\Clamps\.openclaw\workspace-oragorn\artifacts\session_dumps\`
- No OpenClaw global install patch is required.

Repo policy for `memory/YYYY-MM-DD-*.md` within this repo:
- Historical tracked dumps are grandfathered and **not flagged**.
- New/untracked dumps matching this pattern are flagged as `memory-dumps`.
- `.gitignore` ignores new dumps by default:
  - `memory/[0-9][0-9][0-9][0-9]-[0-9][0-9]-[0-9][0-9]-*.md`

Curated memory remains tracked (for example):
- `MEMORY.md`
- `MEMORY-INDEX.md`
- explicitly tracked historical memory files

## Backup Artifact Policy
Context backup files are ignored by default:
- `agents/oragorn/agent/CONTEXT.bak-*.md`

## Lock Staleness Policy
Lock scan root:
- `data/state/**/*.lock`

Stale criteria:
- lock age > `LockStaleMinutes` (default 60), OR
- owner pid can be inferred and process is not running

No automatic deletion is performed by this gate.

### FAIL behavior for stale locks
In `-Mode FAIL`:
- any stale lock causes gate failure
- message is stronger if stale lock includes autopilot lock path:
  - `data/state/autopilot.lock`
  - `data/state/locks/autopilot_worker.lock`

## Runtime Wiring
Autopilot entrypoint:
- `scripts/automation/run_autopilot_task.ps1`

Current default call includes:
- `-RepoHygieneMode FAIL`

Worker supports escape hatch:
- `-SkipRepoHygieneGate`

## Safe override procedure
### Temporary bypass (single run)
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/pipeline/autopilot_worker.ps1 -SkipRepoHygieneGate
```

### Relax to WARN for a run
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/pipeline/autopilot_worker.ps1 -RepoHygieneMode WARN
```

### Re-enable strict mode
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/pipeline/autopilot_worker.ps1 -RepoHygieneMode FAIL
```

## Remediation playbook (non-destructive)
1. Run gate manually.
2. Fix reported groups in order:
   - root-leaks
   - memory-dumps
   - backups
   - stale-locks
   - other
3. For stale locks:
   - confirm owner process truly not active
   - remove lock manually only when safe
4. Re-run gate until `RESULT: CLEAN`.
