# QUANDALF_BRAIN v1 Contract

Purpose: deterministic, model-agnostic long-term memory for Quandalf. This contract defines storage, validation, indexing, and retrieval behavior.

## 1) Directory contract

Canonical root: `brain/`

- `brain/evidence/` → immutable-ish supporting artifacts (notes, csv snapshots, links, screenshots metadata).
- `brain/facts/` → factual claims about markets and outcomes.
- `brain/rules/` → durable strategy/risk heuristics and decision rules.
- `brain/constraints/` → hard limits (risk, venue, execution, compliance, tooling limits).
- `brain/failures/` → known failure modes / anti-patterns.
- `brain/journal/` → concise cycle entries with pointers (Phase A: pointer-only policy).
- `brain/index/` → generated deterministic index (`objects.jsonl`, `journal_tail.json`).
- `brain/working_sets/` → generated retrieval packs for a run scope.

Only `evidence/` and object/journal markdown are edited manually.
`index/` and `working_sets/` are generated by tooling.

## 2) Object file contract

Each object file in `brain/{facts,rules,constraints,failures}` is markdown with YAML frontmatter.

Frontmatter arrays must use multi-line YAML syntax (`- item`), never inline (`[a, b, c]`).

Required frontmatter fields:

- `id` (string, globally unique)
- `type` (`fact|rule|constraint|failure`)
- `title` (string)
- `status` (`active|review|deprecated|superseded`)
- `confidence` (number 0..1)
- `evidence_paths` (array of repo-relative paths)
- `updated_at` (ISO-8601 timestamp)

Recommended fields:

- `tags` (array of short labels; include asset/timeframe when relevant)
- `supporting_ids` (array of object ids)
- `contradictory_ids` (array of object ids)
- `supersedes` (array of object ids)
- `superseded_by` (array of object ids)
- `validated_at` (ISO-8601 timestamp)

Body: concise rationale or interpretation. The source of truth for machine checks is frontmatter.

## 3) Journal policy (Phase A)

Journal entries live in `brain/journal/*.md`.

Expected frontmatter:

- `id`
- `ts`
- `pointers` (array of object ids and/or evidence paths)

Phase A policy: journal should remain pointer-oriented and concise. Violations are WARN (not FAIL) to avoid blocking operations while migrating habits.

## 4) Retrieval contract (working set)

`build_working_set.py` must produce a deterministic JSON pack with:

- run scope metadata (asset/timeframe/tags/scope id)
- primary objects selected for scope
- top supporting objects (`N=3`)
- top contradictory objects (`N=2`)
- applicable constraints and failures
- last 2 journal entries
- latest validation timestamps
- `missing_contradictions` flag
- `warn` indicator (true when contradictions missing or validation warns)

Cap total included objects to 12.

## 5) Revalidation policy

- Run `validate_brain.py` after memory updates.
- Run `build_index.py` after successful updates so retrieval reads fresh index.
- Prefer updating `validated_at` whenever evidence materially changes.
- Revalidate supersession links when creating `supersedes`/`superseded_by` references.

## 6) Validator checks

FAIL conditions:

- invalid/missing required fields per schema
- object `type` mismatches directory type
- missing evidence paths
- unresolved `supersedes` / `superseded_by` links

WARN conditions:

- journal entry violates Phase A pointer-only policy
- unresolved optional links (`supporting_ids`, `contradictory_ids`)
- missing `validated_at`

Output contract:

`BRAIN_VALIDATE [OK|WARN|FAIL] counts=objects:<n>,fail:<n>,warn:<n>`

Exit code contract:

- exit `1` only on FAIL
- exit `0` on OK or WARN
